6//! Framework for testing and benchmarking [`godot-rust`](godot)-dependent code.
7//!
8//! When using the standard `#[test]` Rust macro in a Godot project, tests may fail if they involve objects requiring the Godot
9//! executable to run. `godot-test` addresses this limitation by introducing the [`#[gditest]`](macro@gd_rehearse_macros::gditest) and
10//! [`#[gdbench]`](macro@gd_rehearse_macros::gdbench) macros. These can be used to annotate functions intended for testing or benchmarking
11//! your code, and the crate provides the [`GdTestRunner`] [`GodotClass`](trait@godot::prelude::GodotClass) for executing them within
12//! a Godot scene.
13
14pub use gd_rehearse_defs::runner::GdTestRunner;
